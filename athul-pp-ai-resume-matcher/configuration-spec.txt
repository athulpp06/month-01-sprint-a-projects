Project: AI Resume Matcher
Author: Athul P P

External Services Connected:
1. Google Gemini API (AI Processing)

Authentication Types Used:
1. Google Gemini: API Key (stored in n8n encrypted Credential Manager)

Node Flow & Logic:
- Manual Trigger: The workflow is designed for local execution by clicking "Execute Workflow".
- Data Intake: Uses the "Read Binary File" node to access a PDF resume from the designated secure folder (C:\Users\HP\.n8n-files).
- Parsing: An "Extract from File" node converts the binary PDF data into processable text.
- Job Aggregation: A "Set" (Edit Fields) node holds a hardcoded array of open job requirements to simulate a multi-source data feed.
- AI Logic: The Google Gemini node (using model gemini-1.5-flash) receives the resume text and the job requirements via an expression prompt. It generates the best match and an explanation.
- Output: The AI-generated match is mapped to the final output node for review.

Note: Due to local network constraints preventing secure public webhook callbacks, this workflow is configured for manual local file processing.
